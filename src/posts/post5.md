자바스크립트 처음 배울 때 헷갈리는 게 바로 `var`, `let`, `const` 차이이다.

각 변수마다 **스코프(scope)**, **호이스팅(hoisting)**, **재할당 가능 여부**가 다르다.

오늘은 이 세 가지 기준으로 확실하게 정리해본다.

---

## 1. 기본 개념 정리

| 키워드 | 스코프      | 재선언 | 재할당 | 호이스팅     | 특징                             |
| ------ | ----------- | ------ | ------ | ------------ | -------------------------------- |
| var    | 함수 스코프 | 가능   | 가능   | O (초기화 X) | 오래된 방식, 지금은 잘 안씀      |
| let    | 블록 스코프 | 불가   | 가능   | O (초기화 X) | 일반적인 변수 선언용             |
| const  | 블록 스코프 | 불가   | 불가   | O (초기화 X) | 상수 선언, 참조 자체는 변경 불가 |

---

## 2. 스코프(Scope) 차이

### var → 함수 스코프

```jsx
function test() {
    var x = 1;
    if (true) {
        var x = 2;
    }
    console.log(x); // 2
}
test();
```

var는 블록({}) 무시하고 **함수 단위**로만 범위를 가진다.

그래서 if나 for 안에서 선언해도 같은 스코프 내에서 덮어쓰기 된다.

### let / const → 블록 스코프

```jsx
function test() {
    let x = 1;
    if (true) {
        let x = 2;
        console.log('안쪽:', x); // 2
    }
    console.log('바깥:', x); // 1
}
test();
```

let과 const는 **블록 내부에서만 유효**하다.

같은 이름이어도 블록이 다르면 서로 다른 변수다.

for문이나 if문 내부에서 안전하게 변수 관리할 수 있다.

---

## 3. 재선언 & 재할당

### var — 재선언 가능, 재할당 가능

```jsx
var a = 1;
var a = 2;
a = 3;
console.log(a); // 3
```

같은 이름으로 여러 번 선언 가능해서 실수로 덮어쓸 수 있다.

그래서 요즘은 거의 쓰지 않는다.

### let — 재선언 불가, 재할당 가능

```jsx
let b = 1;
// let b = 2;  SyntaxError
b = 3;
```

### const — 재선언 X, 재할당 X

```jsx
const c = 1;
// c = 2;  TypeError
```

하지만 객체는 참조가 유지되므로 내부 값은 바꿀 수 있다.

```jsx
const obj = { name: 'Jimin' };
obj.name = 'Joji';
console.log(obj); // { name: 'Joji' }
```

const는 **참조 자체를 못 바꾸는 것**이지, 객체 내부 값까지 막는 건 아니다.

---

## 4. 호이스팅(Hoisting)

호이스팅은 **변수 선언이 코드 실행 전에 최상단으로 끌어올려지는 현상**이다.

### var

```jsx
console.log(a); // undefined
var a = 10;
```

var는 선언이 호이스팅되면서 초기값은 undefined가 된다.

그래서 에러 없이 undefined가 출력된다.

### let / const

```jsx
console.log(a); // ReferenceError
let a = 10;
```

let과 const도 호이스팅은 되지만 **초기화 전 접근 금지(TDZ)** 때문에 에러가 난다.

즉 JS 엔진이 “변수는 있지만 아직 사용할 수 없다”라고 판단하는 구간이다.

---

## 5. 실제 사용 가이드

| 상황              | 추천 키워드 | 이유                                        |
| ----------------- | ----------- | ------------------------------------------- |
| 일반 변수         | let         | 재할당 가능, 블록 스코프로 안전             |
| 절대 안 바뀌는 값 | const       | 참조 불변성 보장                            |
| 레거시 코드       | var         | 이미 존재하는 경우만, 새로는 사용하지 말 것 |

기본적으로 **const를 먼저 쓰고, 바꿔야 하면 let**. var는 요즘 거의 안 쓴다.

---

## 6. for문에서 let/var 차이

실무에서 자주 헷갈리는 부분이다.

```jsx
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// 결과: 3 3 3

for (let i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// 결과: 0 1 2
```

var는 한 개의 i를 공유하지만,

let은 반복마다 새로운 i가 만들어진다.

---

## 결론

-   var → 함수 스코프, 재선언 가능, 지금은 거의 안 씀
-   let → 블록 스코프, 재할당 가능
-   const → 블록 스코프, 재할당 불가, 객체 내부 값은 변경 가능

정리하면 **“기본은 const, 바꿀 필요 있으면 let, var는 쓰지 않는다”**

이게 요즘 JS 컨벤션이다.
